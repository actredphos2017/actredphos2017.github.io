# MySQL 的数据类型

## 整数类型
由小到大分为
```sql
- TINYINT   (1字节8位)
- SMALLINT  (2字节16位)
- MEDIUMINT (3字节24位)
- INT       (4字节32位)
- BIGINT    (8字节64位)
```
---
## 浮点数类型
从小到大分为
```sql
- FLOAT         (4字节)
- DOUBLE        (8字节)
- DECIMAL(M,D)  (M+2字节, D为实际显示时保留的小数位数)
```
---
## 日期与时间类型
```sql
- YEAR (1901 ~ 2155)
    使用4位字符串表示时 范围为'1901' ~ '2155'

    使用2位字符串表示时 范围为'00' ~ '99'
        '00' ~ '69' 范围的值会被转换为 2000 ~ 2069 范围的YEAR值
        '70' ~ '99' 范围的值会被转换为 1970 ~ 1999 范围的YEAR值

    使用2位数字表示时 范围为 1 ~ 99
        1 ~ 69 范围的值会被转换为 2001 ~ 2069 范围的YEAR值
        70 ~ 99 范围的值会被转换为 1970 ~ 1999 范围的YEAR值
        
    注意区分 '0' 与 0 的区别, '0' 表示的为 2000, 0 表示的为 0000

- DATE (1000-01-01 ~ 9999-12-3)
    以'YYYY-MM-DD'或者'YYYYMMDD'字符串格式表示

    以'YY-MM-DD'或者'YYMMDD'字符串格式表示
        YY表示的是年, 范围为'00' ~ '99'
        '00' ~ '69'范围的值会被转换为2000 ~ 2069范围的值
        '70' ~ '99'范围的值会被转换为1970 ~ 1999范围的值

    以YY-MM-DD或者YYMMDD数字格式表示

- TIME (-838:59:59 ~ 838:59:59)
    以'D HH:MM:SS'字符串格式表示。
    D表示日, 可以取0 ~ 34之间的值, 插入数据时, 小时的值等于D×24+HH

    以'HHMMSS'字符串格式或者HHMMSS数字格式表示

    使用CURRENT_TIME或NOW()输入当前系统时间

- DATATIME (1000-01-01 00:00:00 ~ 9999-12-31 23:59:59)
    以'YYYY-MM-DD HH:MM:SS'或者'YYYYMMDDHHMMSS'字符串格式表示的日期和时间
    取值范围为'1000-01-01 00:00:00' ~ '9999-12-3 23:59:59'。
    
    以'YY-MM-DD HH:MM:SS'或者'YYMMDDHHMMSS'字符串格式表示的日期和时间
    YY表示年, 取值范围为'00' ~ '99'
    '00' ~ '69'范围的值会被转换为2000 ~ 2069范围的值
    '70' ~ '99'范围的值会被转换为1970 ~ 1999范围的值

    以 YYYYMMDDHHMMSS 或者 YYMMDDHHMMSS 数字格式表示的日期和时间

    使用NOW来输入当前系统的日期和时间

- TIMESTAMP (1970-01-01 00:00:01 ~ 2038-01-19 03:14:07)
    与 DATATIME 大致相同, 但取值范围较小
    使用 CURRENT_TIMESTAMP 来输入系统当前日期和时间
    输入 NULL 时, 系统会输入系统当前日期和时间
    无任何输入时, 系统会输入系统当前日期和时间
```

## 字符串与二进制类型
```sql
- CHAR      (固定长度字符串)
- VARCHAR   (可变长度字符串)
- BINARY    (固定长度二进制数据)
- VARBINARY (可变长度二进制数据)

- TINYBOLB
- BOLB      
- MEDIUMBLOB
- LONGBLOB
(二进制大数据)

- TINYTEXT
- TEXT
- MEDIUMTEXT
- LONGTEXT
(大文本数据)

- ENUM      (枚举类型, 只能存储一个枚举字符串值)
    定义规范:
        ENUM('值1','值2','值3',...,'值n')

- SET       (表示字符串对象, 能存储多个枚举字符串值)
    定义规范:
        SET('值1','值2','值3',...,'值n')

- BIT       (表示位字段类型)
    定义规范:
        BIT(M)
        在上述格式中, M用于表示每个值的位数, 范围为1 ~ 64
        如果分配的BIT(M)类型的数据长度小于M, 将在数据的左边用0补齐

```
